カルーセル

基本構成(自動スライドのみ)

構造
  • カルーセル(外側) div.carousel.slide 属性
    • 属性 data-ride="carousel" で自動スライド(除去するとコントロール移動のみ)
    • カルーセル(内側) div.carousel-inner
      • アイテムは表示順に div.carousel-item 属性
        • 最初に表示するアイテムに .active を設定(ひとつだけ)
        • (オプション) data-interval="{msec}" でスライド表示時間を設定可能
        • スライド画像 img.d-block.w-100 属性
          • 属性 src="..."

折り畳み

ボタンを用いてセレクタで指定した(隠れ)ターゲット要素の表示をON/OFFする。

基本例

設定例: 左は a , 右は button , どちらもその下の div.collapse#ex-collapse をON/OFF

a.btn.btn-primary

div.collaps#ex-collapseAnim pariatur cliche reprehenderit, enim eiusmod high life accusamus terry richardson ad squid. Nihil anim keffiyeh helvetica, craft beer labore wes anderson cred nesciunt sapiente ea proident.

構造
  • 操作ボタンは次のどちらかを用いる
    • a の場合: a.btn.btn-(primary|secondary|...) 属性
      • 属性 data-toggle="collapse"
      • 属性 href="#{ターゲットID}" (ID以外のCSSセレクタも可)
      • アクセシビリティ属性 role="button"
    • button の場合: button.btn.btn-(primary|secondary|...) 属性
      • 属性 type="button"
      • 属性 data-toggle="collapse"
      • 属性 data-target="#{ターゲットID}" (ID以外のCSSセレクタも可)
    • 共通のアクセシビリティ属性 属性
      • aria-expanded="false" (初期状態で表示する場合 "true" )
      • aria-controls="{ターゲットID}"
  • ターゲット(任意要素可) 属性
    • クラス.collapse を設定(デフォルト状態は非表示)
    • 初期状態で表示する場合 .show を追加
    • ターゲットセレクタを設定
      • (例) id="{ターゲットID}"
      • クラスなど任意のCSSセレクタ設定も可
上の ボタンや ボタンも折り畳み機能を利用して作られている。

複数ターゲットの連動ON/OFF

ターゲットセレクタに(IDではなく)クラスを用いることにより複数ターゲットを同時にON/OFFすることができる。

構造
  • 操作ボタンの属性を変更(複数なのでクラスを用いる)
    • a の場合 href=".{ターゲットクラス}"
    • button の場合 data-target=".{ターゲットクラス}"
  • 連動ON/OFFするコンテンツに class="{ターゲットクラス}" を設定

メニューアイテム

(バージョン4から)メニューアイテムとして button.dropdown-item が使える(リンクではないため選択してもデフォルト動作でページ移動しない)。

構造
  • ドロップダウンメニュー div.dropdown-menu 属性
    • アクセシビリティ属性 aria-labelledby=...
    • メニューアイテム button.dropdown-item
      • 属性 type="button"

選択できない(テキスト表示のみの)アイテムを (span).dropdown-item-text で作成できる。またアイテムには任意のHTML要素やCSSを用いた書式設定を行うことができる。

表示のみのアイテム(選択できない)設定は .dropdown-item-text (この表示は code と .text-muted を使用)通常アイテムにも設定可: strong , italic , sub , sup配色も可能: primary secondary success info warning danger

状態表示(アクティブ / 無効)

アイテムに .active を設定するとアクティブ(青背景に白字)表示になる(次の2番目アイテム)。

Regular linkActive linkAnother link

アイテムに .disabled を設定すると灰色表示になる(次の2番目アイテム)。

Regular linkDisabled linkAnother link

ただしどちらも表示効果のみの設定で、.active を用いたドロップダウンで別のアイテムを選択してもアクティブアイテムは変更されない。また.disabled を設定したアイテムも選択可能で、要素が a の場合はページ移動もする(無効化されない)。これは上の2つのボタンで実際に確認することができる。

(応用) JavaScriptによる処理

これらを適切に処理するにはJavaScriptが必要になる。簡単な例を作ってみた。

構造(ポイントのみ) JavaScriptコード
  • ボタンは今までと同じ
  • ドロップダウンメニューにIDを設定 id="active-disabled-example"
  • アイテム1を .active 、アイテム5を.disabled に設定
  • 別の位置にspan#active-indicator で現在のアクティブアイテムを表示
// Dropdown: Active/disabled states demo
$('#active-disabled-example').delegate('a', 'click', function(event) {
  var item = $(this);
  event.preventDefault();                   // prevent default link behaviors
  if (item.hasClass('disabled')) return;    // implement disabled behaviors
  // update the active item
  $('#active-disabled-example a').removeClass('active');
  item.addClass('active');
  $('#active-indicator').text(item.text());
});
  • click イベントハンドラを親(ドロップダウンメニュー)に設定
    • イベントは親が受けて子アイテム a にデリゲート(子アイテムが個別に受けるより効率的)
  • イベントハンドラ
    • まず event.preventDefault() でデフォルト動作を阻止
      • アイテムが a の場合必須 - これがないと(先頭に)ジャンプする
      • アイテムが button の場合は不要
    • 無効化の動作を実装
      • クリックしたアイテムに .disabled が設定されていれば何もしない(return)
    • アクティブ動作を実装
      • まず全アイテムから .active を削除
      • クリックしたアイテムに .active を追加
    • アクティブアイテム表示を更新
