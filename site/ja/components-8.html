モーダルの構成要素

表示サンプル: 常時表示するように変更したもの(通常の初期状態は非表示)

構造
  • 外枠 div.modal.fade 属性
    • クラス .fade - アニメーション効果用(外すと起動時にスライドしない)
    • 属性 id="{モーダルID}" (起動用に必要)
    • 属性 tabindex="-1" (キーフォーカスから外す)
    • アクセシビリティ属性 role="dialog"
    • アクセシビリティ属性 aria-labelledby="{ラベルID}"
    • アクセシビリティ属性 aria-hidden="true"
    • 本体 div.modal-dialog 属性
      • アクセシビリティ属性 role="document"
      • コンテント div.modal-content
        • ヘッダ div.modal-header
          • タイトル h5.modal-title 属性
            • (アクセシビリティ用)属性 id="{ラベルID}"
            • テキスト Modal title
          • 右上クローズボタン button.close 属性
            • 属性 type="button"
            • 属性 data-dismiss="modal"
            • アクセシビリティ属性 aria-label="Close"
            • ボタンシンボル span 属性
              • アクセシビリティ属性 aria-hidden="true"
              • テキスト &times;
        • ボディ div.modal-body
          • (例: 上記サンプルの場合) p
            • テキスト Modal body text goes here.
        • フッタ div.modal-footer
          • Closeボタン button.btn.btn-secondary 属性
            • 属性 type="button"
            • 属性 data-dismiss="modal"
            • テキストClose
          • 用途により追加ボタン(例: 上記サンプルの場合) button.btn.btn-primary 属性
            • 属性 type="button"
            • テキスト Save changes
設定量は多いがボディ内部以外はほとんど同じ定型構造なのでテンプレートを作っておくとよい(このページのpugソースはmixinによるテンプレートを用いている)。

モーダルの起動

モーダルはボタンを使って起動できる。起動ボタンの基本構成は次の通り。

  • button.btn.btn-primary : 表示設定は何でもよいが、次の属性が必須
    • 属性 type="button"
    • 属性 data-toggle="modal"
    • 属性 data-target="#{モーダルID}"
    • テキスト(例) Launch demo modal

次のボタンを押すとページが暗転し、モーダルが上からスライドする。

Modal title

Woohoo, you're reading this text in a modal!

data-target には任意のCSSセレクタ書式が可能。そのため id 以外のセレクタ指定も可能だが、もし複数該当してもモーダルはひとつしか起動しない。起動するモーダルは常に一つで、ネスト(モーダル表示中に別のモーダルを起動)にも対応していない。
動作のポイント(まとめ)
  • 背後の元ページを暗転するため全画面を半透明要素(バックドロップ)で覆う(z-indexは 1040 )
  • その上にモーダルを中央上揃えで表示する(Z-indexは 1050 )
    • .fade を付けると上からスライド(CSSアニメーション)して表示
  • 表示中はキーボードフォーカスやタブストップはモーダルのみが受ける
  • スクロールバーのスクロール動作もモーダルが受ける(次に例を示す)
  • 次のいずれかでクローズして元に戻る
    • クローズボタン(右上シンボル × または右下 Close ボタン)をクリック
    • モーダル外部の背景(バックドロップ)をクリック
    • Esc キー入力

起動時のコンテント更新

同じモーダルの起動時に起動ボタンの情報を読んで表示を変える例(要JavaScript)。

new message
JavaScriptコード 
$('#examplemodal').on('show.bs.modal', function(event) {
  var button = $(event.relatedTarget);
  var recipient = button.data('whatever');
  var modal = $(this);
  modal.find('.modal-title').text('New message to ' + recipient);
  modal.find('.modal-body input').val(recipient);
});
  • 起動ボタンに次の属性を設定(例): data-whatever="@mdo"
  • JavaScript(jQuery)で起動イベント show.bs.modal を検出(⇒ Events ): 次の2つの違いに注意
    • show.bs.modal 起動(ボタンクリック)時: まだ表示前(こちらを使用)
    • shown.bs.modal モーダル表示の完了時
  • 起動ボタン( event.relatedTarget )から属性 data-whatever を読み取る
  • 読み取った値をタイトルと input 要素に設定
