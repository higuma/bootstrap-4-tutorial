Disabled and active states

Pagination items should be disabled or activated with various situations. The most common conditions are as follows.

  • The current page item is displayed as active, but its link is disabled.
  • On showing the first page, Previous item is disabled for display(grayed) and link
  • On showing the last page, Next item is disabled for display(grayed) and link
Setting disabled states

Add .page-item to li . Its descendant link a is disabled automatically.

  • li.page-item
    • Set class .disabled to disabled items
    • Link a
      • Set attribute tabindex="-1" to disabled links (to remove from tabstops)
      • Text (e.g. Previous ) (displays grayed automatically)
      • For accessiblity, use span.sr-only to the current item
        • Text (current)

Setting an active state

Setting .active to li.list-item displays an item as active state (displayed as primary context). But it does not disable an inner link. Furthermore, it cannot be used with .disabled (li.page-item.active.disabled does not work).

This problem can be resolved by changing inner elements from a to span for all disabled items.

  • Active or disabled items li.page-item.(active|disabled)
    • span.page-link (from a.page-link )

Popovers

Popover is a popup component like those found in iOS. It utilizes Popper.js for implementation.

Initialization

Popovers are inactivated as defaut. JavaScript is required to activate them. Below is a simple code to activate all popovers.

$('[data-toggle="popover"]').popover();
Caution: jQuery, Popper.js, and bootstrap.js are required. Use this script after those libraries.

How to use it

  • Available for any elements with setting following attributes
    • data-toggle="popover"
    • title="{title-text}"
    • data-content="{content-text}"
    • One of title or data-content is required. It does not show if both values are empty.

Example (Click to show, click again to hide)

Directions

  • Add an following attribute ( "right" as default)
    • data-placement=("top"|"right"|"bottom"|"left")

Example

Popovers always watch viewport size and elements' posistions, and switch to the opposite sides when space is not enough. It can be demonstrated with the above example by scrolling (top/bottom) or changing viewport width (left/right).

Dismiss on next click

As default, a popover shows on click and hides on the next click on the same element. This specification enables showing multiple popovers at the same time (can be demonstrated by the Directions example above).

However, it is possible to dismiss a popover by clicking another element (more precisely, showing/hiding by focus events) with the following settings.

  • Always use a for the target element (some browsers may not work with button )
  • Set attribute data-trigger="focus"
  • Set attribute tabindex="0"

Using popovers for disabled elements

If the attribute disabled is set to the element, it also disables a popover (example below).

  • button.btn.btn-primary Attributes
    • Attribute type="button"
    • Attribute data-toggle="popover"
    • Attribute title="Disabled popover"
    • Attribute disabled (also disables a popover)
    • Text Disabled button

Example (does not popover on click)

To enable a popover for a disabled element, add a wrapping parent and set popover to it.

  • Wrapping parent span Attributes
    • Utility class .d-inline-block (set to the same display setting as button )
    • Attribute data-toggle="popover"
    • Attribute title="Disabled popover"
    • Target element button.btn.btn-primary Attributes
      • Attribute type="button"
      • Attribute style="pointer-events: none;" (to prevent receiving pointer events)
      • Attribute disabled
      • Text Disabled button
