Regarding accessibility

Navigations ought to have a role navigation. However it cannot be set to ul elements (see W3C - the ul element : navigation is not allowed to be a role value of ul).

You can use parent wrapper elements for ul-based navigations. For this purpose, nav (see W3C - the nav element) is a proper element.

  • Parent wrapper nav (or div, etc.) + Role attribute role="navigation"
    • Navigation ul.nav

    • For the nav-based nav, you can set the roll directly.

    • Navigation nav.nav +Role attribute role="navigation"
    Caution: The role always must be navigation. Even with .nav-tabs, it must not be tab, tablist, or tabpanel.
    It is also recommended to use span.sr-only (⇒ Screenreaders) with text e.g. (current) within an active element for screen reader users. See the actual example in Navbar.

    JavaScript behavior (tabbable panes)

    You can build tabbable panes with a list group (using bootstrap.js) by setting as follows.

    For ul-based nav

    Structure
    • Tablist ul.nav.nav-(tabs|pills) Attribute
      • Accessibility attribute role="tablist" (Caution: not "navigation" )
      • Tab item li.nav-item
        • Link a.nav-link Attributes
          • Add .active to an initlly-visible item
          • Attribute id="{tab-id}"
          • Attribute data-toggle="tab" or "pill" (no difference)
          • Attribute href="#{content-id}"
          • Accessibility attribute role="tab"
          • Accessibility attribute aria-controls="{content-id}"
          • Accessibility attribute aria-selected="true" (on active) or "false"
          • Text (e.g.) Home
    • Tab content div.tabcontent
      • Content item div.tab-pane Attributes
        • Add .fade for fade in/out animation effect
        • Add .show.active to an initially-visible item
        • Attribute id="{content-id}"
        • Accessibility attribute role="tabpanel"
        • Accessibility attribute aria-labelledby="{tab-id}"
