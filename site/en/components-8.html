Structure
  • Modal div.modal Attributes
    • Animation .fade (does not slides down if removed)
    • Attribute id="{modal-id}" (required for invocation)
    • Attribute tabindex="-1" (removes from tab stops)
    • Accessibility attribute role="dialog"
    • Accessibility attribute aria-labelledby="{label-id}"
    • Accessibility attribute aria-hidden="true"
    • Modal dialog div.modal-dialog Attribute
      • Accessibility attribute role="document"
      • Content div.modal-content
        • Header div.modal-header
          • Title h5.modal-title Attribute
            • Accessibility attribute id="{label-id}"
            • Title text Modal title
          • Close button (upper-right) button.close Attributes
            • Attribute type="button"
            • Attribute data-dismiss="modal"
            • Accessibility attribute aria-label="Close"
            • Button symbol span Attribute
              • Accessibility attribute aria-hidden="true"
              • Symbol text &times;
        • Body div.modal-body
          • Content (e.g.) p
            • Text Modal body text goes here.
        • Footer div.modal-footer
          • Close button button.btn.btn-secondary Attributes
            • Attribute type="button"
            • Attribute data-dismiss="modal"
            • Text Close
          • Additional button (e.g.) button.btn.btn-primary Attribute
            • Attribute type="button"
            • Text Save changes
There are so many markups. But most of them are fixed presets (without title, body and additional button(s)). Making a template is convenient to make use of modals. This page utilizes pug mixin templates (see source).

Modal invocation

Modals can be invoked with buttons.

  • button.btn.(btn-primary|...) Following attibutes are required
    • type="button"
    • data-toggle="modal"
    • data-target="#{modal-id}"
    • Text (e.g.) Launch demo modal

See the example below (clicking the button show a modal).

Modal title

Woohoo, you're reading this text in a modal!

Example above finds the modal element by ID. However, data-target accepts arbitrary CSS selector. It is possible to use a selector to query mutiple elements but invokes only the first one. Modals also do not support nesting invocations.
Behaviors (summary)
  • Background is covered by a dark translucent whole-screen element (backdrop) (Z-index = 1040 )
  • Modal shows at top-center (Z-index = 1050 )
    • With .fade , uses CSS animation on showing modal (slides down from top)
  • While displaying modal, keyboard event are sent to modal
  • Modal also get scrolling events (see example below)
  • Modal closes on one of followings
    • Click a close button (upper-right × symbol, or bottom-right Close button)
    • Click on background outside modal (backdrop)
    • Esc key input

Varying modal content

Example below is a demo for displaying button(recipient)-dependent contents to the same modal using JavaScript.

new message
JavaScript code 
$('#examplemodal').on('show.bs.modal', function(event) {
  var button = $(event.relatedTarget);
  var recipient = button.data('whatever');
  var modal = $(this);
  modal.find('.modal-title').text('New message to ' + recipient);
  modal.find('.modal-body input').val(recipient);
});
  • Preliminarily set recipients' data in invocation buttons: e.g. data-whatever="@mdo"
  • Set event handler for show.bs.modal : see Events for the difference of following two events
    • show.bs.modal (used here) On modal triggered (not yet displayed)
    • shown.bs.modal On modal displayed
  • In event handler
    • Get the invocation button event.relatedTarget
    • Read attribute data-whatever to variable recipient
    • Set recipient to the modal title and the element input
